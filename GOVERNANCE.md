# Governance

Qadam Flow is part of **[AI Qadam Build](https://build.aiqadam.org/)** —
the open-source infrastructure layer of AI Qadam. The canonical Build
governance — what Build is, ownership, licensing, contribution model
(DCO), project lifecycle, and the honest governance stage — lives at:

→ **[build.aiqadam.org](https://build.aiqadam.org/#governance)** (summary)
→ **[`GOVERNANCE.md`](https://github.com/aiqadam/build.aiqadam.org/blob/main/GOVERNANCE.md)** (full canonical text)

The text used to live in this file (v0.1, draft). It has been promoted
to canonical (v0.2+) at the location above so it can serve every Build
project from one place, without drift.

## Project-specific governance

Governance that is **specific to Qadam Flow** — this project's
maintainers, scope, contribution mechanics, code of conduct, roadmap —
lives in this repo:

- [README.md](./README.md) — what Qadam Flow is, status, quick start
- [CONTRIBUTING.md](./CONTRIBUTING.md) — how to contribute (DCO sign-off
  is required per the canonical Build governance §5)
- [CODE_OF_CONDUCT.md](./CODE_OF_CONDUCT.md) — community standards
- [ROADMAP.md](./ROADMAP.md) — what's planned

## Why this split

AI Qadam Build is **one** infrastructure layer with **many** projects.
Foundation-level governance (DCO model, ownership rule, money rule,
project stages, etc.) should live in **one place** so the rules are the
same for every Build project and update in lockstep. Each project keeps
only what is unique to it.

This mirrors the model used by neutral foundations like the
[Apache Software Foundation](https://www.apache.org/foundation/governance/)
and [CNCF](https://www.cncf.io/about/charter/), which is the destination
described in [GOVERNANCE §10](https://github.com/aiqadam/build.aiqadam.org/blob/main/GOVERNANCE.md#10-governance-stage-honest).
